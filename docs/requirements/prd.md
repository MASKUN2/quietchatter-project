# QuietChatter Product Requirements Document (PRD)

이 문서는 QuietChatter 서비스의 핵심 가치, 목표 및 기능적 요구사항을 정의합니다.
레거시 프로젝트(quiet-chatter-docs, quiet-chatter-front-end)의 요구사항과 구현 내역을 통합 반영합니다.

## 1. 제품 개요 (Introduction)

- 제품명: QuietChatter (슬로건: "You Belong Here")
- 정의: 내향적인 독자들을 위한 부담 없는 독서 나눔 소셜 네트워크 서비스(SNS).
- 목표: 자신을 드러내기 쑥스러워하는 사람들도 편안하게 책에 대한 이야기를 나누고, 조용히 공감할 수 있는 공간을 제공.

## 2. 핵심 가치 (Core Values)

### 2.1 서비스 철학

- 익명성 (Anonymity): 개인정보를 최소화하고, 부담 없이 자신의 생각을 표현할 수 있다.
- 휘발성 (Ephemerality): 일정 시간(1년)이 지나면 글이 자동으로 숨겨져, 기록에 대한 부담을 줄인다.
- 저자극 (Low Stimulation): 댓글이나 멘션 기능을 배제하고, 단순한 공감 표현만 제공한다.

### 2.2 시스템 가치

- 안정성: 마이크로서비스 분리를 통한 개별 서비스 장애 격리 및 인프라 비용 최적화.
- 편의성: 네이버 도서 API 등 외부 연동을 통한 풍부한 데이터 제공 및 직관적인 UI/UX.
- 보안성: 게이트웨이 기반의 통합 인증 및 회원 식별 정보 보호.

## 3. 사용자 역할 및 권한 (User Roles & Permissions)

사용자는 로그인 여부에 따라 비회원(Anonymous)과 회원(Member)으로 구분된다.

### 3.1 비회원 (Anonymous)

- 도서 검색 및 조회 가능.
- 다른 사람의 북톡(Talk) 읽기 가능.
- 북톡 작성, 반응(좋아요/공감) 등 쓰기 활동은 불가.

### 3.2 회원 (Member)

- 네이버 OAuth 인증을 통한 로그인 필요.
- 북톡(Talk) 작성, 수정, 삭제 가능.
- 공감 표현(좋아요, 공감해요) 가능.
- 내 활동 내역 조회 가능(마이페이지).

## 4. 회원 계정 관리 (Account Management)

### 4.1 계정 상태

- 활성 (Active): 정상적으로 모든 서비스를 이용할 수 있는 상태.
- 탈퇴 (Deactivated): 회원 탈퇴를 선택한 상태.
  - 탈퇴 상태에서는 로그인이 제한된다.
  - 작성한 모든 글은 숨김 처리된다.
  - 탈퇴한 회원이 다시 로그인을 시도할 경우, 계정 재활성화 여부를 묻는 안내가 표시된다.

### 4.2 재활성화 (Reactivation)

- 탈퇴한 회원이 로그인 시도 시, 2시간 유효한 재활성화 토큰(reactivationToken)을 발행한다.
- 사용자가 재활성화에 동의하면 즉시 활성 상태로 복구된다.
- 숨겨졌던 모든 글은 그대로 유지된다(복원되지 않음).
- 토큰 만료 시 재로그인이 필요하다.

## 5. 주요 기능 요구사항 (Functional Requirements)

### 5.1 회원 및 인증 (Member Context)

- 네이버 OAuth 기반의 간편 로그인 및 회원 가입 지원.
- JWT 기반의 인증 토큰 발행 및 쿠키를 통한 세션 관리.
- 미가입 사용자에게 임시 닉네임을 제공하고 회원가입 모달을 통해 닉네임 확정.
- 닉네임 수정 및 회원 탈퇴(비활성화) 기능.

### 5.2 도서 정보 (Book Context)

- 네이버 도서 검색 API를 활용한 실시간 도서 정보 조회.
- 검색 결과는 무한 스크롤(Infinite Scroll) 방식으로 제공.
- 검색된 도서 정보의 로컬 데이터베이스 캐싱 및 조회 성능 최적화.
- 도서 상세 정보 제공: 표지, 제목, 저자, 출판사, 줄거리 등.

### 5.3 도서 탐색 (Book Discovery)

- 홈 화면에서 무작위 알고리즘을 통해 다양한 책 이야기(북톡)를 추천.
- 추천 목록은 일정 간격(45초)으로 자동 갱신되며, 수동 새로고침도 가능.
- 제목, 저자 등의 키워드로 도서 검색 가능.

### 5.4 북톡 및 반응 (Talk & Reaction Context)

#### 5.4.1 북톡 (Book Talk) 작성 규칙 

- 회원은 특정 도서에 대해 최대 250자 이내의 짧은 감상평이나 추천사를 작성할 수 있다.
- 본인이 작성한 글은 수정 및 삭제 가능 (삭제 시 Soft Delete 방식 적용).
- 특정 도서에 대한 북톡은 최신순으로 조회되며 페이지네이션으로 제공.

#### 5.4.2 조용한 공감 (Quiet Reaction) 

- 두 가지 비언어적 반응 방식:
  - 좋아요 (LIKE): 단순한 긍정의 표현.
  - 공감해요 (SUPPORT): 깊은 공감의 표현.
- 반응은 낙관적 업데이트(Optimistic Update) 패턴으로 즉시 화면에 반영.
- 댓글 기능은 제공하지 않으며, 오직 버튼을 통한 비언어적 소통만 가능.

### 5.5 마이페이지 (My Page Context)

- 나의 활동: 회원이 작성한 모든 북톡을 확인 (자동 숨김 날짜 확인 가능, 삭제 가능).
- 프로필 관리: 닉네임 변경 기능.
  - 닉네임 정책: 1~12자의 한글, 영문, 숫자 사용 가능. 공백과 특수문자는 중간에만 허용되며, 시작과 끝에는 사용할 수 없다.
- 회원 탈퇴: 확인 다이얼로그를 통한 탈퇴 처리. 탈퇴 시 작성한 글 모두 숨김 처리.

### 5.6 고객 지원 (Customer Context)

- Voice of Customer(VoC): 사용자가 서비스에 대한 의견을 자유롭게 남길 수 있는 기능.
- 헤더의 VoC 버튼을 통해 모달 형태로 문의 메시지 접수 및 데이터 저장.

### 5.7 서비스 안내 (Service Information)

- 서비스 소개 페이지(About): 핵심 가치, 주요 기능, 프로젝트 연혁 타임라인 제공.
- 이용약관 페이지(Terms of Service) 제공.

## 6. 운영 정책 (Operational Policies)

### 6.1 자동 숨김 정책 (Auto-Hidden Policy)

- 작성 후 1년(기본값)이 지난 북톡(Talk)은 자동으로 숨김된다.
- 사용자는 숨김 시간을 3개월, 6개월, 1년, 3년, 5년, 10년으로 설정할 수 있다.
- 숨김처리된 북톡은 작성한 사용자의 마이페이지에서 조회할 수 있다.
- 사용자는 숨김처리된 북톡을 다시 공개할 수 있다. 
- 숨김처리는 매일 0시를 기준으로 한다.

### 6.2 개인정보 처리

- 서비스 이용을 위해 최소한의 정보(네이버 로그인 식별자 등)만 수집하며, 민감한 개인정보는 저장하지 않는다.
- 회원 탈퇴 시 모든 활동 내역은 식별할 수 없는 상태로 처리된다.

### 6.3 운영 모니터링

- 일간 리포트를 통해 아래 지표를 수집 및 분석:
  - 트래픽 지표: 고유 접속 IP 수, DAU(일간 활성 회원).
  - 안정성 지표: 에러 코드별 발생 건수.
  - 사용자 행동 지표: 인기 검색어, 일간 리액션 현황.
  - 콘텐츠 분석: 조회수 기반 도서 랭킹.

## 7. 비기능적 요구사항 (Non-Functional Requirements)

- 성능: 가상 스레드를 활용한 고성능 I/O 처리 및 JVM 메모리 최적화 정책 준수.
- 확장성: 새로운 도메인 추가 시 독립된 마이크로서비스로 확장이 용이한 구조.
- 신뢰성: 이벤트 기반 아키텍처(EDA)를 통한 서비스 간 데이터 일관성 유지.

## 8. 성공 지표 (Success Metrics)

- API 응답 시간: 게이트웨이 포함 200ms 이내 (외부 API 호출 제외).
- 가용성: 서비스 가동률 99.9% 유지 및 장애 발생 시 데이터 무결성 보장.

## 9. 프론트엔드 UI/UX 요구사항 (Frontend UX Requirements)

### 9.1 페이지 구성

- 홈: 추천 북톡 카드 목록, 자동 갱신 타이머, 업데이트 로그.
- 도서 검색: 키워드 검색, 무한 스크롤 결과 목록.
- 도서 상세: 도서 정보, 북톡 작성 폼, 북톡 목록(페이지네이션), 반응 버튼.
- 마이페이지: 프로필 정보 및 수정, 내 북톡 목록, 회원 탈퇴.
- 프로필 수정: 닉네임 변경 전용 페이지.
- 서비스 소개: 서비스 철학, 주요 기능, 프로젝트 여정 타임라인.
- 이용약관: 서비스 이용약관 안내.

### 9.2 공통 UI 요소

- 공통 헤더: 로고, 검색바, VoC 버튼, 로그인/마이페이지 네비게이션.
- 공통 푸터: 서비스 정보 및 링크.
- 글로벌 오류 바운더리(ErrorBoundary): 렌더링 오류 시 대체 화면 제공.
- 페이지 전환 시 스크롤 초기화(ScrollToTop).
- 토스트 알림(Snackbar): 작업 결과 피드백 제공.

### 9.3 인증 관련 UI 흐름

- 비회원이 쓰기 동작 시도 시 로그인 모달을 표시.
- 네이버 OAuth 콜백 처리 페이지: 로그인 / 회원가입 모달 / 재활성화 모달 분기.
- 최초 방문 사용자를 위한 온보딩 툴팁 제공(자동 숨김 안내 등).
